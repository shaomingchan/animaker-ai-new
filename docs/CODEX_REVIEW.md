**Recommendations**
- 先收敛支付配置：只保留一套服务端 product allowlist，例如 `CREEM_PRODUCT_ID_SINGLE` / `CREEM_PRODUCT_ID_10PACK`，客户端只传 `plan=single|pack`，不要再传裸 productId，也不要在页面和 webhook 里硬编码不同 ID。
- 认证要二选一：要么补齐 `RESEND_API_KEY` 并实际验证 magic link；要么先从 UI 移除 Email 登录入口，只保留 Google。与此同时把 `redirect` / `callbackUrl` 全链路打通。
- i18n 不要再停留在 query 参数 + 首页字典。至少把 `login`、`dashboard`、`create`、`terms`、`privacy`、metadata、`html lang` 全部接入同一套 locale provider，再补一个“所有页面 4 语可切换”的集成测试。
- 把上线前门禁固定成：`npm run lint`、`npx tsc --noEmit`、`next build`。另外清理 `.next` 权限污染，并处理多 lockfile root warning，避免 CI/本地结果不一致。
